# SYNTERRA Development Plan

Date: 2026-05-22
Status: Active roadmap

## Current State

SYNTERRA is no longer only an idea. It now has:

- GitHub repository
- VPS server
- production runtime
- PM2 process manager
- NGINX reverse proxy
- HTTPS / SSL
- Cloudflare DNS
- public domain
- live landing page
- core symbol philosophy

Live domain:

https://synterra-live.com

## Core Architecture Idea

GitHub = memory and source code.
Server = living organism.
Deploy = evolution.
AI = cognition layer.
Users = civilization participants.
Symbol = consciousness center.

## Phase 1: Stabilization

Goal: make the system safe, repeatable and deployable.

Tasks:

1. Create auto deploy system
   - GitHub Actions or webhook deploy
   - git pull on server
   - npm install when needed
   - npm run build
   - pm2 restart synterra

2. Create backup system
   - server backup
   - repository snapshot
   - rollback process

3. Create environment architecture
   - .env.production
   - API key structure
   - runtime configuration

4. Create production logging
   - deploy logs
   - runtime logs
   - error logs

## Phase 2: Core Civilization Interface

Goal: turn the website into a living interface.

Tasks:

1. Integrate SYNTERRA symbol into landing page
   - central animated symbol
   - slow clockwise rotation
   - gold glow
   - one full rotation every 60 to 120 seconds

2. Create app shell
   - sidebar
   - navigation
   - module cards
   - dark/gold visual identity

3. Add symbol watermark system
   - soft background watermark on secondary pages
   - low opacity
   - unified civilization field feeling

4. Make landing buttons functional
   - ENTER links to /app
   - DOCUMENTATION links to /docs
