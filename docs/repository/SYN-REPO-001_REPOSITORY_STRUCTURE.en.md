# Purpose

This document defines the long-term repository structure for SYNTERRA.

The repository is designed to preserve:
- documentation;
- knowledge;
- audits;
- source code;
- platform interface;
- media assets;
- historical archive.

---

# Core Principle

```text
Repository structure reflects system architecture.
```

Folders are not random storage. They represent the system layers.

---

# Main Areas

```text
docs
knowledge
audits
src
gateway
assets
tests
archive
```

---

# Documentation Area

The documentation area stores foundation, architecture, planning, synchronization and specification documents.

---

# Knowledge Area

The knowledge area stores concepts, terminology, graphs, embeddings and metadata.

---

# Source Area

The source area stores runtime, memory, AI, translation and platform implementation code.

---

# Archive Area

The archive area stores legacy, deprecated and historical material.

---

# Main Repository Sentence

> The SYNTERRA repository is the structural memory of a developing civilization-scale system.
